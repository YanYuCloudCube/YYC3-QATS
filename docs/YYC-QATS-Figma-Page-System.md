## 二、页面布局规范（全局统一+模块适配）
### 1. 全局基础布局（12栅格系统，适配所有页面）
```plaintext
flowchart TD
    A[全局容器 1920*1080] --> B[顶部导航区 高度60px 固定]
    A --> C[主体区域 高度calc(100vh-60px)]
    C --> D[左侧导航区 宽度240px 固定/可收缩]
    C --> E[主内容区 宽度calc(100%-240px) 自适应]
    E --> F[面包屑区 高度40px]
    E --> G[操作栏区 高度50px]
    E --> H[核心内容区 高度calc(100%-90px) 滚动]
    A --> I[右侧悬浮面板 宽度80px 固定 可收起]

```
- 顶部导航区：高度60px，固定吸顶，包含logo+主导航+全局操作，无滚动；
- 左侧导航区：默认宽度240px，可收缩至200px/纯图标模式（宽度60px），固定，无滚动；
- 主内容区：基于12栅格，边距16px，内距24px，核心内容区超出高度时纵向滚动，无横向滚动；
- 右侧悬浮面板：宽度80px，固定在页面右侧（距离右16px，距离下24px），可收起为圆形图标（直径60px）。
### 2. 模块专属布局适配
|模块名称|布局特殊适配|核心设计要点|
|-|-|-|
|市场数据中心|主内容区分「左侧品种列表（2栅格）+右侧图表/数据（10栅格）」|图表占比≥70%，筛选面板折叠在右侧抽屉，不占用核心可视区|
|智能策略引擎|策略编辑页分「左侧组件库（3栅格）+右侧画布（9栅格）」|画布区域无边界，支持全屏，代码编辑器占满画布区，语法高亮|
|风险管控中台|实时风控页分「左侧资产列表（3栅格）+右侧风控看板（9栅格）」|风控看板多维度卡片布局，预警卡片置顶，红色闪烁，视觉优先|
|量子计算实验室|资源监控页分「左侧任务列表（4栅格）+右侧算力监控（8栅格）」|算力监控仪表盘占核心区，任务列表支持批量操作，状态颜色区分|

|分辨率|布局调整规则|组件适配|

## 三、核心交互逻辑（按用户操作流程拆解）
### 1. 导航交互逻辑
```plaintext
graph LR
    A[用户点击顶部主导航] --> B{是否当前模块？}
    B -->|是| C[无操作，保持当前页面]
    B -->|否| D[切换至对应模块首页，保留左侧导航展开状态]
    E[用户点击左侧二级导航] --> F{是否当前页面？}
    F -->|是| G[无操作]
    F -->|否| H[加载对应子页面，左侧导航选中态切换，面包屑更新]
    I[用户折叠左侧导航] --> J[导航宽度收缩至60px，仅显示图标，hover显示文字]
    J --> K[用户展开左侧导航] --> L[恢复240px宽度，显示文字+图标]

```
- 核心规则：模块内跳转无全局刷新，仅更新主内容区；跨模块跳转保留用户操作状态（如筛选条件、图表周期）。
### 2. 行情页面核心交互逻辑
```plaintext
graph TD
    A[用户进入全球行情页] --> B[加载默认市场（股票）TOP20品种，3s/次实时刷新]
    B --> C[用户点击品种] --> D[跳转至K线分析页，加载该品种默认周期（1d）K线]
    D --> E{用户操作}
    E -->|切换周期| F[刷新K线数据，保留指标叠加状态]
    E -->|叠加指标| G[实时渲染指标，与K线联动]
    E -->|缩放/平移| H[实时调整K线显示范围，数据点实时加载]
    E -->|导出数据| I[弹出导出配置弹窗，选择格式（PNG/CSV/PDF），确认后下载]

```
- 核心规则：K线数据加载采用“懒加载”（仅加载可视区数据），大数据量时抽样展示，缩放后加载精准数据。
### 3. 策略编辑&回测交互逻辑
```plaintext
graph TD
    A[用户进入策略编辑页] --> B{选择编辑模式}
    B -->|图形化| C[加载组件库+空白画布，默认显示新手引导]
    B -->|代码| D[加载代码编辑器+默认模板，语法高亮]
    C --> E[用户拖拽组件+连线] --> F[实时校验流程合法性，错误即时提示]
    D --> G[用户编写代码] --> H[实时语法校验，错误行红色标记]
    E/G --> I[用户保存策略] --> J[策略存入草稿，提示保存成功]
    J --> K[用户点击智能回测] --> L[弹出回测参数配置弹窗]
    L --> M[用户确认参数] --> N[显示回测进度条，后台执行回测，实时返回核心指标]
    N --> O[回测完成] --> P[生成回测报告，显示收益曲线/最大回撤等，支持导出]

```
- 核心规则：策略编辑实时校验，回测任务支持后台运行（可切换页面，回测完成后弹窗提示）。
### 4. 交易操作核心交互逻辑
```plaintext
graph TD
    A[用户进入实盘交易页] --> B[加载账户资产（1s/次刷新）+持仓列表+委托记录]
    B --> C[用户点击“买入”] --> D[弹出买入配置弹窗（品种/价格/数量/类型）]
    D --> E[用户确认买入] --> F[二次确认弹窗（显示买入信息+风险提示）]
    F --> G[用户确认] --> H[提交委托，实时更新委托记录状态（待成交/成交/失败）]
    H --> I{委托状态}
    I -->|成交| J[更新持仓列表+可用资金，弹出成交提示框]
    I -->|失败| K[弹出失败提示框，显示失败原因]
    B --> L[用户设置止盈止损] --> M[提交条件单，加入交易计划列表，实时监控触发条件]
    M --> N[条件触发] --> O[自动提交委托，按上述流程执行]

```
- 核心规则：所有交易操作二次确认，委托状态实时刷新，条件单触发即时执行，异常即时提示。
### 5. 风控预警交互逻辑
```plaintext
graph TD
    A[系统实时监控风控指标] --> B{指标是否超标？}
    B -->|否| C[持续监控，无操作]
    B -->|是| D[触发预警流程]
    D --> E[实时更新风控看板（红色闪烁）]
    D --> F[弹出预警弹窗（置顶，不可关闭，仅可确认）]
    D --> G[发送预警通知（邮件/短信，按用户配置）]
    E --> H[用户查看预警] --> I{选择操作}
    I -->|手动风控| J[弹出操作弹窗（平仓/减仓/对冲），二次确认后执行]
    I -->|自动风控| K[按预设规则自动执行风控操作，记录操作日志]
    J/K --> L[风控操作完成] --> M[更新风控指标，解除预警状态，记录日志]

```
- 核心规则：预警弹窗置顶，必须确认后关闭；自动风控操作有日志记录，支持追溯。
## 四、关键交互细节（提升用户体验）
1. 数据加载交互：所有异步请求（数据查询/任务执行）均显示加载态，无“空白等待”；大数据量加载时先显示骨架屏，再逐步渲染数据；
2. 操作反馈交互：所有点击操作（按钮/链接/图标）均有视觉反馈（波纹/高亮/缩放），反馈时长100ms，不影响操作流畅性；
3. 快捷键交互：核心操作支持快捷键（如F5刷新、Enter确认、ESC关闭弹窗、Ctrl+S保存），在操作区标注快捷键提示；
4. 错误处理交互：操作错误时即时显示红色提示框（位置：操作按钮下方），提示内容简洁明确（如“委托失败：可用资金不足”），支持点击“查看详情”；
5. 保存状态交互：用户的筛选条件、图表配置、面板布局等操作，自动保存到本地缓存，返回页面时恢复上次状态；
6. 大屏监控交互：支持全屏展示（F11），数据5s/次刷新，支持多屏联动（主屏幕操作同步至副屏幕），异常指标点击可钻取详情。
### 总结
1. Figma文件结构：按「设计系统-组件库-页面原型-交互原型」四层架构搭建，命名规范统一，组件复用性高，便于团队协作；
2. 页面布局：基于12栅格的全局统一布局+模块专属适配，支持多分辨率响应式，核心内容区优先展示，无冗余布局；
3. 交互逻辑：围绕量化交易“高效、精准、实时”核心需求设计，交易/风控操作二次确认，数据加载无空白，操作状态自动保存，异常即时预警，符合专业交易者操作习惯。
