## 核心设计规范前置（UI色彩/阴影视觉/响应交互）
### 一、UI色彩体系（金融专业+科技感，适配量化交易场景，无视觉疲劳）
#### 1. 主色调（品牌/核心交互）
- 深空蓝主色：#0A192F（顶部导航/左侧导航/主按钮，代表专业、稳定）
- 科技蓝浅调：#112240（导航hover/卡片标题栏，主色渐变，层次区分）
- 科技银辅助：#CCD6F6（文字主色/图标常规态，高对比度，易读性优先）
#### 2. 功能色（业务逻辑/状态区分，金融行业通用认知）
- 安全绿（涨/成功/运行中）：#38B2AC（行情上涨/任务完成/交易成功）
- 警示红（跌/危险/异常）：#F56565（行情下跌/风控预警/系统异常）
- 提醒黄（待处理/预警/中风险）：#ECC94B（挂单待成交/中风险提示/待审核）
- 中性蓝（信息/普通提示）：#4299E1（系统提示/数据说明/普通通知）
#### 3. 中性色（页面基底/文字/分割，层级清晰）
- 页面底色：#071425（全局背景，低亮度减少视觉疲劳，适配长时间盯盘）
- 卡片底色：#112240（主内容区卡片，与背景形成弱对比，不刺眼）
- 浅灰底色：#1A2B47（表单/筛选面板，卡片内二级区分）
- 文字层级：
    - 一级标题：#FFFFFF（18-24px，导航/页面标题）
    - 二级文字：#CCD6F6（14-16px，正文/数据标签）
    - 三级文字：#8892B0（12px，辅助说明/占位符/失效数据）
- 分割线/边框：#233554（弱分割，不破坏页面整体性）
### 二、阴影视觉规范（轻质感/无冗余，科技风扁平化阴影，突出层级不浮夸）
#### 1. 阴影层级（4级，适配不同组件，统一模糊/偏移规则）
- 0级（基础）：box-shadow: none（纯文字/分割线/基础按钮）
- 1级（卡片基础）：box-shadow: 0 2px 8px 0 rgba(0,0,0,0.1)（主内容区普通卡片/列表项）
- 2级（悬浮/高亮）：box-shadow: 0 4px 12px 0 rgba(0,0,0,0.15)（卡片hover/选中态/筛选面板）
- 3级（弹窗/抽屉）：box-shadow: 0 8px 24px 0 rgba(0,0,0,0.2)（弹窗/右侧功能抽屉/模态框）
- 4级（全局浮层）：box-shadow: 0 12px 32px 0 rgba(0,0,0,0.25)（登录框/全局提示浮层/大屏监控模块）
#### 2. 阴影通用规则
- 所有阴影均为深灰色系，无彩色阴影，贴合科技风；
- 组件hover/选中时，阴影从低层级升级一级，配合轻微缩放（transform: scale(1.01)），强化交互反馈；
- 弹窗/抽屉的遮罩层为rgba(7,20,37,0.7)（与页面底色一致，半透明，不突兀）。
### 三、响应交互规范（高频操作优先/反馈即时/适配量化交易「高效盯盘+快速操作」需求）
#### 1. 全局交互规则
- 反馈即时性：所有点击/悬停操作100ms内给出反馈，无延迟；
- 加载状态：所有异步操作（数据请求/任务执行/训练）必须显示加载态（骨架屏/加载动画/进度条），无“空白等待”；
- 操作确认：高危操作（平仓/删策略/改配置/清空数据）必须二次确认，普通操作（查询/筛选/收藏）直接执行；
- 页面跳转：模块内跳转用无刷新路由（局部更新），跨模块跳转保留页面状态（返回可恢复）。
#### 2. 核心组件交互
- 导航类：左侧导航hover变背景色+文字高亮，选中态加左侧竖线（3px solid #38B2AC），支持折叠/展开（折叠后显示图标，hover显示文字）；
- 图表类：支持放大/缩小/平移（鼠标滚轮+拖拽），数据点悬停显示详情浮层，点击钻取下一级数据，支持图表切换（K线/分时/柱状图）；
- 按钮类：主按钮（蓝底白字）hover加深背景色，次按钮（透明边框）hover变背景色，危险按钮（红底）hover加深，禁用态置灰（透明度0.5）+不可点击；
- 表单类：输入框聚焦加边框高亮（2px solid #4299E1），校验错误即时提示（红色文字+图标），支持一键清空/批量填充。
#### 3. 响应式适配（核心适配19201080，兼容1366768/2560*1440）
- 1920*1080（主流）：完整显示所有组件，左侧导航宽度240px，主内容区自适应；
- 1366*768（小屏）：自动收缩左侧导航（宽度200px），隐藏非核心面板，筛选器折叠为“更多”按钮；
- 2560*1440（大屏）：主内容区居中，两侧留空，卡片宽度自适应，不拉伸变形；
- 所有页面无横向滚动条，纵向滚动条为自定义样式（贴合页面风格）。
#### 4. 快捷操作交互
- 悬浮面板：右侧悬浮面板（固定位置），支持收起/展开，点击快捷按钮直接执行操作，hover显示说明；
- 右键菜单：仅支持图表/数据点/策略/模型，右键显示快捷操作（无冗余选项），点击空白处关闭；
- 快捷键：支持自定义快捷键（如F5刷新/Enter确认/Ctrl+C复制），在“系统配置-快捷键”中可修改，所有快捷键有提示。

## 三、Figma设计落地要求
1. 文件结构规范：按「设计系统-组件库-页面原型」分层，设计系统包含色彩/字体/阴影/间距，组件库为可复用组件，页面原型按模块分类；
1. 组件复用性：所有组件均为组件实例，修改组件库可同步更新所有页面，无重复设计；
1. 标注规范：所有页面/组件均做精准标注（尺寸/间距/颜色/字体），适配前端开发，无模糊标注；
1. 交互原型：核心页面（行情/策略/交易/风控）制作交互动效原型，导出为Figma原型链接，支持前端查看交互逻辑；
1. 命名规范：所有页面/组件/图层命名统一（英文+数字，无中文），按模块/功能分类，便于协作。
